🛡️ v3.4.15: 스마트 가격 수정 로직 완성 & 불필요한 다이얼로그 제거

🎯 핵심 개선사항:
- 과도한 손실 방지: 가격차이가 최소마진보다 클 때 자동으로 수정 제외 (예: 7,000엔 > 500엔)
- 정확한 수정 필터링: 현재가 적정, 손실 예상 상품은 수정 과정에서 자동 건너뛰기
- 불필요한 다이얼로그 제거: 가격차이가 0이거나 100엔 미만인 상품의 수정 다이얼로그 완전 차단

🔧 스마트 필터링 로직:
- products_to_update 필터링에 상태 조건 추가: '가격 수정 필요' 상품만 수정 대상
- price_difference == 0 조건 명시적 추가로 동일가 상품 완전 제외
- 가격관리 탭과 주력상품 탭 동일한 판단 기준 적용

💡 직관적 상태 표시:
- "✅ 현재가 적정 (동일가)": 가격차이가 정확히 0인 경우
- "✅ 현재가 적정 (최저가)": 내 상품이 최저가인 경우
- "✅ 현재가 적정": 100엔 이내 차이인 경우
- "⚠️ 손실 예상": 과도한 손실이 예상되는 경우

🛠️ UI 안정성 강화:
- 주력상품 통합처리 완료 후 버튼 상태 정상 복원
- restore_ui_signal.emit()으로 UI 상태 복원 보장
- 존재하지 않는 ui_enable_signal 제거

⚖️ 통일된 판단 로직:
1. price_difference <= 0 → 최저가이거나 더 저렴 (수정 안함)
2. abs(current_price - suggested_price) < 100 → 현재가와 제안가 차이 미미 (수정 안함)
3. price_difference == 0 → 가격차이 정확히 0 (수정 안함)
4. price_difference > min_margin → 과도한 손실 예상 (수정 안함)
5. 나머지 → 가격 수정 필요 (수정 진행)

🎉 사용자 경험 완성:
- 불필요한 확인 다이얼로그 완전 제거로 작업 효율성 극대화
- 과도한 손실 자동 방지로 수익성 보호
- 명확한 상태 구분으로 직관적인 의사결정 지원
- 가격관리와 주력상품 탭 간 일관된 동작으로 혼란 방지
