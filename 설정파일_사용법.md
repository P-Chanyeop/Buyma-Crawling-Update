# 설정 파일 사용법 가이드

## 📁 설정 파일 개요

`config_template.json` 파일은 BUYMA 자동화 프로그램의 모든 설정을 관리하는 템플릿 파일입니다.

## 🚀 빠른 시작

### 1. 설정 파일 복사
```bash
# 템플릿 파일을 실제 설정 파일로 복사
cp config_template.json config.json
```

### 2. 기본 설정 수정
`config.json` 파일을 열어 다음 항목들을 수정하세요:

```json
{
  "account_settings": {
    "email": "your-email@example.com",
    "password": "your-password"
  },
  "crawling_settings": {
    "default_crawl_count": 20
  },
  "price_analysis_settings": {
    "default_discount_amount": 100
  }
}
```

## ⚙️ 주요 설정 항목 설명

### 🔐 계정 설정 (account_settings)
```json
{
  "email": "BUYMA 로그인 이메일",
  "password": "BUYMA 로그인 비밀번호",
  "remember_login": true
}
```

### 🔍 크롤링 설정 (crawling_settings)
```json
{
  "default_crawl_count": 20,        // 기본 크롤링 개수
  "timeout_seconds": 10,            // 페이지 로딩 대기 시간
  "retry_count": 3,                 // 실패 시 재시도 횟수
  "delay_between_requests": 3       // 요청 간 대기 시간 (초)
}
```

### 💰 가격 분석 설정 (price_analysis_settings)
```json
{
  "default_discount_amount": 100,   // 기본 할인 금액 (엔)
  "min_margin_yen": 500,           // 최소 마진 (엔)
  "exclude_loss_products": true,    // 손실 상품 제외 여부
  "auto_price_update": true        // 자동 가격 수정 여부
}
```

### 📤 업로드 설정 (upload_settings)
```json
{
  "max_images_upload": 10,         // 최대 업로드 이미지 수
  "include_images": true,          // 이미지 포함 여부
  "skip_duplicate_products": true, // 중복 상품 건너뛰기
  "default_category": "레디스 패션" // 기본 카테고리
}
```

## 🛠️ 고급 설정

### 🌐 네트워크 설정
```json
{
  "network_settings": {
    "connection_timeout": 30,       // 연결 타임아웃 (초)
    "max_redirects": 5,            // 최대 리다이렉트 횟수
    "proxy_settings": {
      "use_proxy": false,          // 프록시 사용 여부
      "proxy_host": "",            // 프록시 호스트
      "proxy_port": 0              // 프록시 포트
    }
  }
}
```

### 🔧 에러 처리 설정
```json
{
  "automation_settings": {
    "error_handling": {
      "max_retries": 3,            // 최대 재시도 횟수
      "retry_delay_seconds": 5,    // 재시도 간 대기 시간
      "auto_recovery_enabled": true, // 자동 복구 활성화
      "create_error_reports": true   // 에러 리포트 생성
    }
  }
}
```

### 🖥️ 시스템 리소스 설정
```json
{
  "system_settings": {
    "performance": {
      "max_memory_usage_percent": 80, // 최대 메모리 사용률
      "max_cpu_usage_percent": 90,    // 최대 CPU 사용률
      "resource_check_interval": 60   // 리소스 체크 간격 (초)
    }
  }
}
```

## 📝 설정 파일 관리

### 설정 백업
```bash
# 현재 설정을 백업
cp config.json config_backup_$(date +%Y%m%d).json
```

### 설정 복원
```bash
# 백업에서 설정 복원
cp config_backup_20250116.json config.json
```

### 기본값으로 초기화
```bash
# 템플릿에서 새로운 설정 파일 생성
cp config_template.json config.json
```

## ⚠️ 주의사항

### 보안 주의사항
1. **비밀번호 보안**: `config.json` 파일에는 실제 비밀번호가 저장되므로 보안에 주의
2. **파일 권한**: 설정 파일의 읽기 권한을 적절히 설정
3. **버전 관리**: Git 등에 업로드 시 비밀번호 정보 제외

### 설정 변경 시 주의사항
1. **JSON 형식**: 올바른 JSON 형식을 유지해야 함
2. **데이터 타입**: 숫자, 문자열, 불린 값의 타입 확인
3. **필수 항목**: 필수 설정 항목은 삭제하지 말 것

## 🔍 문제 해결

### 설정 파일 오류
```bash
# JSON 형식 검증
python -m json.tool config.json
```

### 설정 적용 안됨
1. 프로그램 재시작
2. 설정 파일 경로 확인
3. 파일 권한 확인

### 기본값 사용
설정 파일이 없거나 오류가 있으면 프로그램 내장 기본값이 사용됩니다.
